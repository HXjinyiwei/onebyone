# 审核功能优化实施报告

## 项目概述
根据用户需求，对小说发布与讨论论坛项目的审核功能进行了全面优化，实现了完整的审核流程用户体验。

## 用户需求分析
用户提出了三个核心问题：
1. 首页小说排行榜是否可以显示封面图片
2. 小说名称是否需要重复检查，如果重复是否显示警告
3. 管理后台中的审核管理功能：审核不通过时有什么提示，能否重新提交，是否有优化空间

针对第三个问题，用户补充了详细需求：
- 管理员点击拒绝按钮后弹出提示框输入拒绝原因
- 用户在个人中心"我的消息"中能看到拒绝理由
- 用户可以在"我的小说/帖子"页面点击重新提交
- 进入被保存的修改页面，修改后重新提交

## 技术架构

### 数据库设计
```
novel表新增字段：
- reject_reason: varchar(500) - 拒绝原因

post表新增字段：
- reject_reason: varchar(500) - 拒绝原因

chapter表新增字段：
- reject_reason: varchar(500) - 拒绝原因
```

### API端点
```
# 小说审核
POST /novel/reject-with-reason/{id} - 拒绝小说并指定原因
POST /novel/resubmit/{id} - 重新提交小说

# 帖子审核
POST /post/reject-with-reason/{id} - 拒绝帖子并指定原因
POST /post/resubmit/{id} - 重新提交帖子
```

### 状态流转
```
小说审核状态：
0: 待审核 (pending)
1: 已通过 (approved)
2: 已拒绝 (rejected)
3: 已封禁 (banned)

帖子状态：
0: 正常 (normal)
1: 审核中 (auditing)
2: 已拒绝/已封禁 (rejected/banned)
```

## 完整审核流程

### 管理员端流程
1. 管理员进入管理后台 (`/admin`)
2. 查看待审核的小说/帖子列表
3. 点击"拒绝"按钮
4. 弹出模态框，输入拒绝原因
5. 提交后，系统：
   - 更新小说/帖子的状态为"已拒绝"
   - 保存拒绝原因到数据库
   - 创建消息通知给作者

### 用户端流程
1. 用户登录后进入个人中心 (`/profile`)
2. 在"我的消息"中查看审核拒绝通知
3. 在"我的小说"或"我的帖子"中看到被拒绝的项目
4. 点击"重新提交"按钮
5. 进入编辑页面，看到拒绝原因提示
6. 修改内容后提交
7. 系统自动将状态改为"待审核/审核中"
8. 等待管理员重新审核

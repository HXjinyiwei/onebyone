# 界面优化与评论交互改进方案

本方案旨在根据用户需求，优化发帖与小说编辑页面的选项，并改进帖子详情页评论点赞的实时交互体验。

## 1. 广场发帖页面优化
**目标**：移除 [`post-create.html`](biyeshiji/src/main/resources/templates/post-create.html) 中未实现或冗余的“发布选项”。

### 修改内容：
- 移除 HTML 中的 `<div class="publish-options">` 区域（包含“立即发布”、“保存草稿”单选框以及“锁定帖子”复选框）。
- 确保在提交表单时，如果没有这些字段，后端能正确处理（通常默认为发布状态）。

## 2. 小说创建与编辑页面优化
**目标**：移除 [`novel-create.html`](biyeshiji/src/main/resources/templates/novel-create.html) 中不常用的“草稿”和“锁定”状态。

### 修改内容：
- 在 `<select name="status">` 中删除以下两行：
  - `<option value="2" ...>草稿</option>`
  - `<option value="3" ...>锁定</option>`
- 仅保留“连载中”和“已完结”两个状态。

## 3. 评论点赞实时显示改进
**目标**：在 [`post-detail.html`](biyeshiji/src/main/resources/templates/post-detail.html) 中，点击评论点赞按钮后，立即更新界面显示的数字，而无需刷新页面。

### 修改方案：
- **逻辑调整**：修改 `window.likeComment` 函数。
- **具体实现**：
  1. 在点赞请求成功（`res.code === 200`）后，通过 JavaScript 获取当前评论项的点赞数字。
  2. 根据点赞状态（通常后端会返回是“点赞”还是“取消点赞”的操作结果），动态加 1 或减 1。
  3. 更新对应的 DOM 元素内容。

```javascript
// 示例修改逻辑
window.likeComment = function(commentId) {
    // ... ajax 请求 ...
    success: function(res) {
        if (res.code === 200) {
            layer.msg(res.message, {icon: 1});
            // 实时更新显示
            var $btn = $('[data-comment-id="' + commentId + '"]').find('.tieba-reply-btn').filter(function() {
                return $(this).text().indexOf('点赞') > -1;
            });
            var likeCountText = $btn.text();
            var currentCount = parseInt(likeCountText.match(/\d+/)[0]);
            // 假设 res.data 返回 true 表示点赞成功，false 表示取消点赞
            var newCount = res.data ? currentCount + 1 : currentCount - 1;
            $btn.html('<i class="layui-icon layui-icon-praise" style="font-size: 12px; margin-right: 3px;"></i> 点赞 (' + newCount + ')');
        }
    }
}
```

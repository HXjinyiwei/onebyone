## 2. 实时更新机制设计

### 2.1 轮询方案（推荐）
```javascript
// 核心逻辑
var messagePollingInterval = null;

function startMessagePolling(userId) {
    // 立即获取一次
    fetchUnreadMessageCount(userId);
    
    // 每30秒轮询一次
    messagePollingInterval = setInterval(function() {
        fetchUnreadMessageCount(userId);
    }, 30000);
}

function stopMessagePolling() {
    if (messagePollingInterval) {
        clearInterval(messagePollingInterval);
        messagePollingInterval = null;
    }
}
```

### 2.2 优化考虑
1. **频率控制**：30秒间隔平衡实时性和性能
2. **网络优化**：使用轻量级JSON响应
3. **错误处理**：网络错误时自动重试
4. **资源清理**：页面卸载时停止轮询

### 2.3 消息操作实时反馈
在个人中心页面，消息操作后立即更新红点：
- 标记单条消息为已读 → 红点数字-1
- 标记所有消息为已读 → 红点消失
- 删除消息 → 根据消息状态更新红点

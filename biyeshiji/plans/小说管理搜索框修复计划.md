# 小说管理搜索框修复计划

## 问题描述
在管理后台的小说管理页面中，搜索框点击没有反应，无法搜索。

## 问题分析
经过详细分析，发现了以下问题：

### 1. 前端与后端API参数不匹配
- **前端代码** (`admin.html` 第2004行)：发送 `params.search` 参数
- **后端API** (`NovelController.java` 第140-149行)：期望 `keyword` 参数

### 2. 具体代码位置
- **前端文件**: `biyeshiji/src/main/resources/templates/admin.html`
- **函数**: `loadContentNovels(searchKeyword)` (第1998-2027行)
- **问题代码**: 
  ```javascript
  if (searchKeyword && searchKeyword.trim() !== '') {
      params.search = searchKeyword.trim();  // 错误：应该是 params.keyword
  }
  ```

### 3. 后端API接口
```java
@GetMapping("/all")
public Response<List<Novel>> getAllNovels(
        @RequestParam(required = false) String keyword,  // 期望 keyword 参数
        @RequestParam(required = false) Integer status,
        @RequestParam(required = false) Long categoryId,
        @RequestParam(required = false, defaultValue = "1") Integer page,
        @RequestParam(required = false, defaultValue = "10") Integer pageSize) {
    // ...
}
```

## 修复方案

### 1. 前端代码修改
**文件**: `biyeshiji/src/main/resources/templates/admin.html`
**修改位置**: 第2004行
**修改内容**:
```diff
if (searchKeyword && searchKeyword.trim() !== '') {
-    params.search = searchKeyword.trim();
+    params.keyword = searchKeyword.trim();
}
```

### 2. 验证其他类似问题
检查管理后台中其他搜索功能是否存在相同问题：
- 帖子管理搜索 (`loadContentPosts` 函数)
- 章节管理搜索 (`loadContentChapters` 函数)
- 用户管理搜索 (`loadUsers` 函数)

### 3. 测试验证步骤
1. 启动应用服务器
2. 登录管理后台

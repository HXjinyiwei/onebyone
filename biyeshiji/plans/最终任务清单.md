# 最终任务清单

## 项目概述
基于用户的需求分析，需要完成以下三个主要功能的改进：

## 任务一：首页排行榜增加封面显示
### 目标
在首页的小说排行榜中显示小说封面图片

### 技术方案
1. 修改 `index.html` 模板，在排行榜部分添加封面图片显示
2. 更新 CSS 样式，确保封面图片布局美观
3. 确保封面图片路径正确（使用 `coverImage` 字段）

### 涉及文件
1. `src/main/resources/templates/index.html`
2. `src/main/resources/static/css/common.css`（可选）

### 工作量评估
- 简单，1-2个文件修改
- 低风险，不影响核心功能

## 任务二：小说名称重名检查功能
### 目标
在创建小说时检查是否有重名，如有则提示用户

### 技术方案
1. 在数据库层面添加唯一约束（可选）
2. 在服务层添加重名检查逻辑
3. 在前端添加实时检查或提交时验证
4. 提供友好的错误提示

### 涉及文件
1. `NovelService.java` 和 `NovelServiceImpl.java` - 添加检查逻辑
2. `NovelController.java` - 修改创建API
3. `novel-create.html` - 添加前端验证
4. 数据库：可能需要添加唯一索引

### 工作量评估
- 中等，3-5个文件修改
- 中风险，需要处理数据一致性和用户体验

## 任务三：审核管理功能优化（包含小说、帖子、章节）
### 目标
1. 审核不通过时存储拒绝原因
2. 用户可以看到拒绝原因并重新提交
3. 优化审核流程和用户体验

### 技术方案（采用方案一：各自表添加字段）
1. **数据库修改**：
   - `novel` 表添加 `reject_reason` TEXT 字段
   - `post` 表添加 `reject_reason` TEXT 字段  
   - `chapter` 表添加 `reject_reason` TEXT 字段

2. **后端修改**：
   - 三个实体类添加 `rejectReason` 字段
   - 三个服务接口和实现类修改拒绝方法
   - 三个控制器修改拒绝API接受原因参数
   - 消息服务增强，发送审核通知

3. **前端修改**：
   - `admin.html` 添加拒绝原因输入模态框
   - `profile.html` 显示被拒绝内容和重新提交按钮
   - 三个创建页面添加重新提交逻辑
   - JavaScript处理新的交互逻辑

### 涉及文件（详细清单）
#### 数据库（3个表）

2. post表 - 添加reject_reason字段
3. chapter表 - 添加reject_reason字段

#### 实体类（3个文件）

#### 服务接口（3个文件）

#### 服务实现（3个文件）

#### 控制器（3个文件）

#### 前端模板（4-5个文件）

## 实施优先级建议

### 方案A：按功能优先级
1. **首页封面显示**（简单，快速见效）
2. **重名检查功能**（中等，提升数据质量）
3. **审核功能优化**（复杂，需要仔细测试）

### 方案B：按技术依赖性
1. **审核功能优化**（需要数据库修改，影响范围大）
2. **重名检查功能**（依赖小说相关代码）
3. **首页封面显示**（独立，可随时实施）

### 方案C：混合实施
1. **第一阶段**：首页封面显示 + 审核功能数据库修改
2. **第二阶段**：重名检查功能 + 审核功能后端
3. **第三阶段**：审核功能前端 + 测试优化

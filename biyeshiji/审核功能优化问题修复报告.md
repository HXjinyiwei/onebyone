# 审核功能优化问题修复报告

## 问题概述
用户反馈审核功能存在两个问题：
1. 点击重新提交按钮提示服务器错误
2. 点击编辑后再点击发布帖子提示帖子更新失败

## 问题分析

### 1. 重新提交功能服务器错误
**问题原因**：前端调用了`/novel/resubmit/{id}`和`/post/resubmit/{id}`API端点，但控制器中没有这些端点。

**解决方案**：
- 在`NovelController.java`中添加`/novel/resubmit/{id}`端点（第344-356行）
- 在`PostController.java`中添加`/post/resubmit/{id}`端点（第284-296行）
- 对应的服务层方法已在`NovelServiceImpl.java`和`PostServiceImpl.java`中实现

### 2. 帖子更新失败问题
**问题原因**：`post-create.html`中编辑模式使用了PUT方法，但`PostController.java`中的`updatePost`方法只支持POST方法。

**解决方案**：
- 修改`post-create.html`第380行，将编辑模式的HTTP方法从PUT改为POST
- 同时修改`novel-create.html`第582行，将编辑模式的HTTP方法从PUT改为POST，保持一致性

## 修复详情

### 1. 重新提交API端点修复
**文件修改**：
- `biyeshiji/src/main/java/com/example/biyeshiji/controller/NovelController.java`：
  ```java
  @PostMapping("/resubmit/{id}")
  public Response<Void> resubmitNovel(@PathVariable Long id) {
      try {
          boolean result = novelService.resubmitNovel(id);
          if (result) {
              return Response.success("小说已重新提交，等待审核", null);
          } else {
              return Response.error("重新提交失败");
          }
      } catch (RuntimeException e) {

      }
  }
  ```
